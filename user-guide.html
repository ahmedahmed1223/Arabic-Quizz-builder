1

2
اختيار القسم
يختار الفريق قسماً من شاشة الأقسام (أو بالعجلة في نمط عجلة الحظ). يظهر عدد الأسئلة المتبقية. الأقسام المكتملة تُظهر "✅".
3
تشغيل السؤال
يظهر السؤال مع العداد. للكشف اضغط "🔍 كشف". لانتقال للسؤال التالي اضغط "التالي ←" أو Space/Enter. للبحث عن سؤال معين والانتقال إليه مباشرة اضغط 🔍 (انظر قسم البحث). بعد كشف الإجابة، يتحدد سلوك الانتقال تلقائياً أو يدوياً حسب الإعدادات (انظر قسم الانتقال).
4
منح النقاط
بعد الإجابة يظهر تلقائياً إذا كانت صحيحة أو خاطئة مع نقاط الفريق. يمكن منح نقاط يدوية لأي فريق. في V8: نظام السلاسل يمنح مكافآت تلقائية.
5
العودة للوحة التحكم
اضغط ⚙️ من أي شاشة. التقدم محفوظ تلقائياً. للخروج من ملء الشاشة اضغط Esc أو ⛶.

معلومات العرض
💾
تفاصيل localStorage
يعرض عدد المفاتيح المخزّنة، الحجم الكلي بالكيلوبايت، وقائمة بأكبر 10 مفاتيح مع أحجامها التفصيلية. يساعد في تحديد البيانات التي تستهلك مساحة كبيرة.
🗄️
تفاصيل IndexedDB
يعرض عدد العناصر المخزّنة، الحجم الكلي التقريبي، ومفاتيح IDB المستخدمة. بما أن الوسائط الكبيرة تُخزّن حصرياً في IndexedDB، فهذا القسم مهم لمراقبة حجم الصور والملفات الصوتية.
📏
معلومات الحصة
يعرض الحجم المستخدم والمتاح والنسبة المئوية من حصة التخزين الكلية. يُنبّهك عندما تقترب من الحد الأقصى لاتخاذ إجراء (مثل حذف وسائط قديمة أو تنظيف البيانات).
✅
فحص البيانات الأساسية
يفحص حالة البيانات الأساسية (Primary State) في IndexedDB ويعرض عدد الأقسام والأسئلة المحفوظة. إذا وُجدت فئات بدون أسئلة، يُستعاد تلقائياً من IndexedDB.
أدوات التنظيف
🔍
فحص التخزين

يفحص صحة التخزين بالكامل ويعرض تقريراً مفصلاً. يكشف البيانات التالفة أو المتضاربة ويقترح حلولاً.

🗑️
تنظيف البيانات اليتيمة

يحذف البيانات اليتيمة في IndexedDB التي لم تعد مرتبطة بأي قسم أو سؤال. يساعد في استعادة المساحة المهدرة دون التأثير على البيانات الفعّالة.

⚠️في وضع التصفح الخاص، قد لا يتوفر IndexedDB. يعمل مراقب التخزين في الوضع العادي فقط. إذا لاحظت بطءاً في التطبيق، افحص مراقب التخزين — كثرة الوسائط الكبيرة قد تسبب بطئاً ملحوظاً.

نظام الإشعارات الذكي V12
تنبيهات ذكية تُبقيك على اطلاع دائم
🔔يُقدّم الإصدار 12 نظام إشعارات ذكياً يُبقيك على اطلاع بأحداث المنصة المهمة دون إزعاج. يراقب النظام عدة مصادر للتنبيهات ويعرضها بشكل غير مفاجئ مع شريط تقدم مرئي. يمكنك تخصيص أنواع الإشعارات التي تريد تلقيها من الإعدادات.
📚
إشعارات تحديثات المكتبة
عند ربط مكتبة خارجية، يُرسل النظام إشعاراً عند توفر تحديثات جديدة. يُعرض الإشعار بعدد الأسئلة الجديدة والمعدّلة والمحذوفة مع زر لمراجعة التغييرات وتطبيقها.
🔋
تنبيهات البطارية
عند انخفاض مستوى البطارية عن 20%، يظهر تنبيه يُنصح بحفظ العمل الحالي. هذا يحمي من فقدان البيانات بسبب انطفاء الجهاز فجأة أثناء المسابقة.
📡
تنبيهات مشاكل الاتصال
إذا كان التطبيق يعتمد على مصدر خارجي (مثل المكتبة الخارجية أو Google Sheets)، يُنبّهك عند فقدان الاتصال بالإنترنت وعند استعادته، مع عرض حالة المزامنة.
❓
تحذيرات نقص الأسئلة
عندما يكون عدد الأسئلة في قسم ما أقل من المطلوب للمسابقة، يظهر تحذير يُنصح بإضافة أسئلة أو تقليل عدد الأسئلة المطلوبة. هذا يمنع نفاد الأسئلة أثناء المسابقة.
📊
شريط تقدم الإشعارات
عند إجراء عمليات طويلة (استيراد، تصدير، مزامنة)، يظهر شريط تقدم مرئي يُعرض حالة العملية والوقت المتبقي التقريبي. هذا يمنع القلق أثناء العمليات الطويلة.
💡يمكنك التحكم بأنواع الإشعارات من لوحة التحكم → الإعدادات → الإشعارات. يمكنك تعطيل أي نوع من الإشعارات بشكل مستقل.

صور الخيارات (Option Images)
رفع صورة مستقلة لكل خيار من خيارات الإجابة
🖼️إلى جانب نوع السؤال "صوري" الذي يعرض صورة فوق السؤال، يوفّر التطبيق ميزة صور الخيارات المستقلة التي تتيح رفع صورة مختلفة لكل خيار من خيارات الإجابة (أ، ب، ج، د). هذه الميزة تعمل مع جميع أنواع الأسئلة النصية والصورية والرياضية.
كيفية إضافة صورة لخيار
1
أنشئ أو حرّر سؤالاً
افتح نافذة إنشاء أو تحرير السؤال من أي قسم. اختر نوع السؤال (نصي، صوري، رياضي، إلخ). ميزة صور الخيارات متاحة لمعظم أنواع الأسئلة التي تحتوي خيارات متعددة.
2
اضغط زر الصورة 🖼️ بجانب الخيار
بجانب كل حقل خيار (أ، ب، ج، د) يوجد زر صغير 🖼️. اضغطه لفتح نافذة اختيار ملف الصورة. الصيغ المدعومة: JPG، PNG، GIF. الحد الأقصى لحجم الصورة: 3 ميغابايت.
3
معاينة وإزالة
بعد رفع الصورة تظهر معاينة مصغّرة بجانب الخيار. لإزالة الصورة اضغط زر ✕ على المعاينة. يمكن الجمع بين نص وصورة في نفس الخيار.
💡 لا يشترط وجود نص مع الصورة — يكفي الصورة وحدها لاعتبار الخيار صالحاً

أنماط العرض التقديمي الثلاثة
⚡
مباشر (direct) — الافتراضي
يُعاد مباشرة إلى شاشة اختيار الأقسام دون أي شاشات وسيطة. الخيار الأسرع والأبسط. مناسب للمسابقات السريعة والجلسات غير الرسمية.
▶️
تسلسلي تلقائي (sequence)
يتدرج تلقائياً عبر الشاشات بالترتيب: ترحيب ← شكر ← فرق ← أقسام. تنتقل كل شاشة للأخرى تلقائياً بعد 5 ثوانٍ. مناسب للعروض التي تريد فيها لفت انتباه الجمهور لكل عنصر.
🖐️
تسلسلي يدوي (manual)
نفس تسلسل الشاشات لكن الانتقال يتم يدوياً بالضغط على زر "التالي ←" أو مفتاحي ← →. يمنح المقدم تحكماً كاملاً في توقيت كل شاشة. مثالي عندما يكون هناك حديث أو تعليق بين كل شاشة.

تنبيه بدون خصم (warn) — الافتراضي

تظهر نافذة بخيارات ثلاثة: ➕ إضافة N نقطة (للإجابة الشفهية الصحيحة)، ➖ خصم N نقطة (للخطأ)، و⏭ تجاوز. قيمتا الإضافة والخصم قابلتان للتخصيص من الإعدادات (الافتراضي: 1 نقطة). هذا الوضع هو الأكثر مرونة ويتيح للمقدم تسجيل النقاط يدوياً حسب تقديره.

💡للمسابقات التنافسية الرسمية يُنصح بوضع "إضافة/خصم" لأنه يمنح المقدم تحكماً كاملاً. للمسابقات التعليمية المفتوحة يكفي وضع "تنبيه". استخدم "صامت" فقط إذا كنت واثقاً من تحكمك.

البحث أثناء اللعب

1

2
ابحث بالنص أو الرقم

3
انتقل للسؤال المطلوب
اضغط على أي سؤال في نتائج البحث للانتقال إليه مباشرة. الأسئلة المستخدمة سابقاً تظهر بعلامة ✅ وشفافية مخفّضة لتمييزها عن الأسئلة الجديدة. يمكن العودة لأي سؤال حتى لو تمت الإجابة عليه.

سؤال الترتيب — دليل تفصيلي
كيفية الإنشاء والعرض والتقييم
📋سؤال الترتيب هو نوع متقدم يطلب من المتسابق إعادة ترتيب مجموعة من العناصر بالترتيب الصحيح. يدعم 3 طرق للتفاعل: السحب والإفلات، أزرار الأعلى والأسفل، والسحب باللمس على الجوال.
إنشاء سؤال ترتيب
1
اختر نوع "ترتيب"
عند إنشاء سؤال جديد، اختر تبويب "📝 ترتيب" من أنواع الأسئلة. يظهر حقل إدخال العناصر المرتبة.
2
أدخل العناصر بالترتيب الصحيح
اكتب العناصر بالترتيب الصحيح من الأعلى للأسفل. هذا هو الترتيب الذي يجب على المتسابق إعادة تكوينه. مثال: رتب الأحداث التاريخية من الأقدم للأحدث.
3
ضبط باقي الخصائص
حدد الصعوبة والوقت وشرح الإجابة كما في أي سؤال آخر. يمكنك أيضاً تفعيل/تعطيل "إظهار أزرار صح/خطأ لأسئلة الترتيب" من الإعدادات.

مثال: بناء وحدة تحكم مخصصة
💡يمكنك بناء وحدة تحكم خاصة بك تتواصل مع المنصة عبر BroadcastChannel. كل ما تحتاجه هو فتح نفس الصفحة في نافذة أخرى والانضمام للقناة quiz_remote_v1.
const channel = new BroadcastChannel('quiz_remote_v1'); function sendCommand(action) { channel.postMessage({ action, ts: Date.now() }); } channel.onmessage = (e) => { if (e.data.action === 'state_update') { const s = e.data.payload; console.log('الفرق:', s.teams); console.log('السؤال:', s.currentQ); console.log('الوقت:', s.timeLeft + '/' + s.timerTotal); } }; sendCommand('ping'); // الأوامر المتاحة: // sendCommand('next'); // سؤال تالي // sendCommand('reveal'); // كشف الإجابة // sendCommand('categories'); // شاشة الأقسام // sendCommand('scores'); // شاشة النتائج // sendCommand('podium'); // شاشة المنصة // sendCommand('timer'); // تشغيل/إيقاف المؤقت // sendCommand('music'); // تشغيل/إيقاف الموسيقى
